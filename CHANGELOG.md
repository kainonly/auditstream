# Changelog

所有对 AuditStream 项目的重要更改都将记录在此文件中。

本文档格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，
并且本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [3.0.0] - 2026-01-19

### 新增 (Added)

#### 核心功能
- 全新微服务架构，支持模块化部署和独立扩展
- 事件驱动机制，实现组件间松耦合通信
- 插件系统，支持自定义审计规则和处理器动态加载
- 流式处理引擎，支持大规模审计日志实时分析
- 智能审计规则配置系统
- 实时审计流监控和告警功能
- 完整的审计链路追踪和回溯能力
- 内置合规性报告模板（SOC2、ISO27001等）

#### API
- RESTful API v3，提供完整的审计流管理接口
- GraphQL 支持，提供灵活的数据查询能力
- Webhook 集成，支持审计事件自动推送
- API 版本管理机制，确保向后兼容

#### 数据处理
- 多数据源统一审计（数据库、文件系统、云存储等）
- 内置敏感数据脱敏功能
- 自动化审计数据归档和清理策略
- 多级缓存策略，优化查询性能

#### 用户界面
- 全新的审计数据可视化仪表板
- 自定义审计报表和视图
- 明暗主题切换支持

#### 安全性
- OAuth2.0 和 SAML 认证支持
- TLS 1.3 数据传输加密
- 审计日志签名验证机制
- 基于 RBAC 的细粒度权限控制

### 变更 (Changed)

- 优化并发处理模型，性能提升 300%
- 改进错误处理机制，提供更详细的错误信息
- 升级日志系统，支持结构化日志输出
- 重构代码结构，提升代码可读性和可维护性
- 内存占用优化，降低 50%
- 平均响应时间从 150ms 降至 45ms
- 审计日志处理速度从 10K/s 提升至 50K/s
- 并发处理能力从 1000 req/s 提升至 4000 req/s

### 已弃用 (Deprecated)

- API v1.x 已弃用，将在 v4.0 中移除
- 旧版配置文件格式（v2.x）将在 v4.0 中不再支持

### 移除 (Removed)

- 移除对 Node.js 16.x 以下版本的支持
- 移除旧版同步处理引擎
- 移除已过时的审计规则配置方式

### 修复 (Fixed)

- 修复高并发场景下的内存泄漏问题
- 修复审计日志丢失的边界条件
- 修复配置热重载导致的状态不一致问题
- 修复权限验证的安全漏洞

### 安全 (Security)

- 加强身份认证机制，防止未授权访问
- 修复 SQL 注入漏洞
- 修复 XSS 攻击漏洞
- 增强审计日志的防篡改能力

### 破坏性变更 (Breaking Changes)

- **最低 Node.js 版本**: 从 16.x 升级到 18.x
- **数据库要求**: PostgreSQL 12+ 或 MySQL 8.0+
- **Redis 版本**: 最低要求 6.0+
- **API 端点**: v1 API 已移除，需迁移到 v3
- **配置格式**: 配置文件结构已重新设计
- **数据库架构**: 表结构有重大调整，需执行迁移脚本
